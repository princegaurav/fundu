What Fundu Does

Seven guarantees, every task.

Completeness Check

New

The most common AI failure: touching 2 of 8 files and calling it done. Before the evidence bundle, Fundu cross-references git diff --name-only against the planned file list, runs grep to confirm zero stale references remain, and the same model that implemented must verify — no delegation. If it's not in the ledger, it didn't happen.

Adversarial Review

Core

All reviewers launch in parallel. No fixes happen until every verdict is in — acting on partial results causes churn. After all models respond, findings are deduplicated, conflicts resolved, and everything fixed in a single pass. Medium tasks: 1 reviewer. Large tasks or security-sensitive files: GPT + Gemini + Claude Opus, simultaneously.

SQL Evidence Ledger

Core

Every build, test, lint, completeness check, and reviewer verdict is INSERTed into a SQLite table during the run. The evidence bundle at the end is a SELECT — not prose the model writes. If the row isn't there, the check didn't run. No more hallucinated results.

Baseline Snapshots

Core

Before a single file changes, Fundu snapshots your build, tests, and diagnostics into the ledger. After implementation, it runs everything again and compares. Regressions surface immediately — before you see the code, not after you ship it.

Pushback

Core

Fundu is a senior engineer, not an order-taker. Bad approach? Unnecessary tech debt? Dangerous edge case? It says so before writing a single line — at both the implementation and the requirements level. You can override it. But you'll know you did.

Session Memory

Core

Broke a file last session? Fundu remembers. Learned your test command? Won't ask again. Knows which files are high-risk from prior failures? Automatically escalates those to Large. Context that actually persists across conversations.

Git Autopilot

Core

Stashes uncommitted work, creates a branch off main before touching anything — you're never editing trunk. After verification passes, commits with a clear message and shows you the exact git revert command to undo everything in one line.

Under The Hood

Every task runs the same loop. No shortcuts.

  1. Understand & Recall Parses your request, checks session history for prior work on the same files — including what failed and why.
  2. Survey & Pushback Finds existing patterns to reuse. Evaluates whether the approach makes sense at both requirements and implementation level. Pushback surfaces before any code is written.
  3. Baseline Snapshot Build, tests, and diagnostics all recorded into SQL. This is the "before" state — every regression comparison starts here.
  4. Implement Surgical, minimal changes following your codebase's existing patterns. Extends what's there rather than inventing new abstractions.
  5. Build → Tests → Lint All exit codes and outputs INSERTed into SQL. No pass/fail claim is accepted without a row in the ledger.
  6. Completeness Check Cross-references git diff against the planned file list. Runs grep for stale references. The same model that implemented must sign off — no delegation. Gate: 3 of 4 minimum checks for Medium, 4 of 4 for Large.
  7. Adversarial Review All reviewer agents launch in parallel. Zero fixes until every verdict is in. Findings deduplicated, conflicts resolved, then fixed in one pass. Max 2 review rounds.
  8. Evidence Bundle & Commit Report generated via SELECT from the SQL ledger — not written by the model. Auto-commit with clear message and rollback command shown.

Proportional Effort

Verification scales with risk.

Not every task needs three AI models. Fundu picks the right level of scrutiny automatically — and any file touching auth, payments, crypto, or CI/CD is always treated as Large.

Size When Completeness
Check		 Adversarial
Reviewers
Tiny Read-only, docs, explanation — —
Small Single file, low risk change — —
Medium Bug fix, feature, refactor ✓ 1 model
Large Multi-file, new architecture, risk files ✓ 3 models (parallel)
